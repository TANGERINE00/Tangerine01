                     ------------------------         PROYECTO  TANGERINE         ------------------------

BIENVENIDOS AL PROYECTO TANGERINE DE DESARROLLO DEL SOFTWARE, SECCIÓN 2. ESTE PROYECTO SE BASA EN UN SISTEMA QUE PRETENDE
AUTOMATIZAR TODA LA GESTIÓN DE CLIENTES DE CUALQUIER COMPAÑIA DE DESARROLLO.

LAS FUNCIONALIDADES SE ENCUENTRAN DIVIDIDAS EN 10 MÓDULOS QUE REALIZAN LAS SIGUIENTES TAREAS:
1. MÓDULO DE INGRESO, SEGURIDAD Y DASHBOARD INICIAL.
2. MÓDULO DE USUARIOS Y ROLES.
3. MÓDULO PARA LA GESTIÓN DE LEADS.
4. MÓDULO PARA LA GESTIÓN DE COMPAÑÍAS.
5. MÓDULO PARA LA GESTIÓN DE CONTAXTOS POR COMPAÑÍAS O POR LEADS.
6. MÓDULO PARA LA GESTIÓN DE PROPUESTAS POR COMPAÑÍAS.
7. MÓDULO PARA LA GESTIÓN DE PROYECTOS POR COMPAÑÍAS.
8. MÓDULO PARA LA GESTIÓN DE FACTURAS POR PROYECTOS.
9. MÓDULO PARA LA GESTIÓN DE PAGOS POR FACTURAS.
10. MÓDULO PARA LA GESTIÓN DE EMPLEADOS.

USTED PODRÁ EMPEZAR A UTILIZAR EL PROYECTO DESDE UN AMBIENTE DE DESARROLLO, COMO POR EJEMPLO, VISUAL STUDIO. PARA ESTO
LO PRIMERO QUE DEBERA HACER ES CLONAR EL REPOSITO DE MANERA LOCAL, ES DECIR, A TRAVÉS DE GITHUB DEBERA CLONAR EL
REPOSITORIO Tangerine01. USTED ESCOGERA LA CARPETA EN DONDE DESEA QUE SE GUARDEN LOS ARCHIVOS.

UNA VEZ CLONADO USTED PODRÁ ABRIR EL PROYECTO EN VISUAL STUDIO A TRAVÉS DE LA SIGUIENTE RUTA: ARCHIVO -> ABRIR ->
PROYECTO O SOLUCIÓN -> TANGERINE. LUEGO DE REALIZAR ESTE PROCEDIMIENTO, DEBERA AGREGAR LOS PROYECTOS A LA SOLUCIÓN.

PARA AGREGAR LOS PROYECTOS HAGA CLICK DERECHO EN LA SOLUCIÓN Y SELECCIONE AGREGAR, LUEGO PROYECTO EXISTENTE Y SELECCIONARA
EL ARCHIVO LLAMADO DatosTangerine DENTRO DE UNA CARPETA CON EL MISMO NOMBRE. ESTE MISMO PROCEDIMIENTO DEBERA REPETIRLO
PARA: DominioTangerine, ExcepcionesTangerine, LogicaTangerine, PruebasUnitarias, Tangerine_Contratos y 
Tangerine_Presentador.

CADA UNA DE ESTAS SON LAS CAPAS DEL PROYECTO, ESTAS CAPAS CONTIENEN LA SIGUIENTE INFORMACIÓN:

DatosTangerine: CONTIENE LOS DAO DE ACCESO A DATOS PARA CADA UNO DE LOS MÓDULOS, ESTA CAPA SE ENCARGA DE LA 
LÓGICA DE ACCESO A DATOS.

DominioTangerine: CONTIENE CADA UNA DE LAS ENTIDADES USADAS POR CADA MÓDULO.

ExcepcionesTangerine: CONTIENE EL MANEJO DE EXCEPCIONES QUE CADA MÓDULO SE ENCARGA DE APLICAR EN SU LÓGICA DE NEGOCIOS.

LogicaTangerine: CONTIENE LOS COMANDOS QUE CADA MÓDULO UTILIZA PARA EL MANEJO DE SU LÓGICA DE NEGOCIOS.

Tangerine: CONTIENE LAS VISTAS DEL PROYECTO.

Tangerine_Contratos: CONTIENE LA INTERFAZ DE CADA UNA DE LAS VISTAS.

Tangerine_Presentador: CONTIENE EL PRESENTADOR DE CADA UNA DE LAS VISTAS.

PruebasUnitarias: CONTIENE LAS PRUEBAS DE CADA UNO DE LOS MÉTODOS IMPLEMENTADOR POR CADA MÓDULO.
